# AI漫画制作テンプレート

このフォルダを Codex で開き、**やりたいことを日本語で話しかける**だけで、
企画（Step0）から完成ページ画像・表紙（Step8）まで順番に作れます。

## 使い方（3ステップ）

1. このフォルダをまるごとダウンロードし、好きな名前にリネームします（例: `my-manga`）。
2. Codex でこのフォルダを開きます。
3. 「漫画の企画を考えたい」と話しかけて始めます。あとは Codex が案内します。

## 最初に触るファイル

- **`config.yaml`** … 作品タイトル・絵柄・章一覧。まずここだけ埋めればOK。
- **`characters.yaml` / `locations.yaml`** … 登場人物・場所の名簿。Step3 で埋めます（最初は空でOK）。

## 作っていくと増えるフォルダ（読む順に番号付き）

| フォルダ | 中身 |
|----------|------|
| `step0_企画/` | 企画（Step0） |
| `step1_構成最適化/` | 目次・章構成の最適化（Step1） |
| `step2_脚本/` | 脚本（Step2） |
| `step3_設定資料/` | キャラ・場所の設定資料と参照画像（Step3） |
| `step4_ページ設計/` | ページ単位の設計図（Step4） |
| `step5_コマ構成/` | コマ割り・演出・セリフ（Step5） |
| `step6_yaml生成/` | 画像生成用YAML（Step6） |
| `step7_完成画像/` | 完成ページ画像（Step7） |
| `step8_表紙/` | 表紙（Step8） |

## 触らなくてよいもの

- `AGENTS.md` … Codex への指示書（中身を読む必要はありません）。
- `scripts/` … Codex が必要なときに自動で使う部品。自分でコマンドを打つ必要はありません。

## 進め方の流れ

Step0 企画 → Step1 構成 → Step2 脚本 → Step3 設定資料＋参照画像 →
Step4 ページ設計 → Step5 コマ構成 → Step6 YAML生成 → Step7 画像生成 → Step8 表紙作成

困ったら「今どこまで進んだ？」「次は何をすればいい？」と Codex に聞いてください。

## GitHubでバージョン管理したい人向け（任意）

進捗管理にGitHubを使いたい場合は、このフォルダで `git init` してそのままpushして構いません
（Gitは前提にしていませんが、使っても支障はありません）。

- 作品データなので**Privateリポジトリ**を推奨します。
- `step3_設定資料/` や `step7_完成画像/`・`step8_表紙/` に画像が増えるとリポジトリ容量が大きくなります。
  気になる場合は画像を Git LFS で管理してください（必須ではありません）。
